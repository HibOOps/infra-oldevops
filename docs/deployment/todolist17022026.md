## Modifications futures à planifier

### Priorité haute — Stabilité du pipeline

#### 1. Migrer le CI Runner hors du Proxmox homelab
**Problème** : CT 210 (ci-runner) est géré par le même Terraform qu'il exécute.
Un `terraform apply` ou `rollback.sh` peut le détruire et couper le workflow.

**Plan** :
- Provisionner un VPS OVH/Hetzner dédié au runner
- Configurer WireGuard VPN entre le VPS et le homelab (accès SSH aux containers)
- Re-enregistrer le runner GitHub Actions sur le VPS
- Supprimer `module "ci_runner"` de Terraform (géré hors-bande)

#### 2. Extraire `module "ci_runner"` en resource directe avec `prevent_destroy`
**Problème** : `lifecycle { prevent_destroy = true }` ne peut pas s'appliquer à un bloc `module`, uniquement à un `resource`.

**Plan** :
```hcl
# Remplacer dans main.tf :
resource "proxmox_lxc" "ci_runner" {
  ...
  lifecycle {
    prevent_destroy = true
    ignore_changes  = all
  }
}
```

#### 3. Ajouter la clé SSH du runner dans le bootstrap Ansible de CT 210
**Problème** : Si CT 210 est recréé, la clé SSH du runner (`id_ed25519`) sera vide → Ansible échoue.

**Plan** :
- Stocker la clé privée du runner dans Ansible Vault ou GitHub Secret
- Ajouter une tâche dans `ansible/roles/github-runner/tasks/main.yml` pour écrire la clé au bootstrap

#### 4. Augmenter le wait post-rollback selon les services
**Problème** : 45s peut ne pas suffire pour que tous les services Docker redémarrent (Grafana, PostgreSQL).

**Plan** :
- Implémenter un wait-with-retry dans `rollback.sh` (boucle avec `curl` jusqu'à réponse)
- Ou augmenter à 90s avec un commentaire explicatif

### Priorité moyenne — Qualité et maintenabilité

#### 5. Corriger l'inventaire Ansible (monitoring affiché sur 192.168.1.250 au lieu de .202)
**Problème** : Les logs Ansible affichaient le mauvais host pour le playbook monitoring.

**Plan** : Vérifier `ansible/inventory.ini` et corriger les groupes `[monitoring]` et `[app_demo]`.

#### 6. Supprimer `node_modules` du dépôt git
**Problème** : `app-demo/backend/node_modules` et `app-demo/frontend/node_modules` sont commités,
alourdissant le repo et causant les problèmes de `copy` Ansible.

**Plan** :
```bash
echo "app-demo/*/node_modules/" >> .gitignore
git rm -r --cached app-demo/backend/node_modules app-demo/frontend/node_modules
git commit -m "chore: remove node_modules from git tracking"
```

#### 7. Protéger la branche `main` via PR obligatoire (sans bypass admin)
**Problème** : Tous les fixes ont été pushés directement sur `main` (bypass admin).
En production réelle, chaque fix devrait passer par une PR.

**Plan** :
- Créer des branches `fix/*` pour chaque correction
- Merger via PR après validation CI
- Désactiver le bypass admin une fois le pipeline stable
